RFC 3986 & WHATWG URL API

Read RFC →
Legacy Approach PHP < 8.5 
$components = parse_url("https://php.net/releases/8.5/en.php");

var_dump($components['host']);
// string(7) "php.net"
Modern Solution PHP 8.5 
use Uri\Rfc3986\Uri;

$uri = new Uri("https://php.net/releases/8.5/en.php");

var_dump($uri->getHost());
// string(7) "php.net"
Why it matters

New Uri\Rfc3986\Uri and Uri\WhatWg\Url classes provide standards-compliant URL parsing. Say goodbye to edge cases and inconsistencies—robust handling of internationalized domains, query parameters, and fragments built right in.

02

Clone with Property Modification

Read RFC →
The Old Way PHP < 8.5 
final readonly class PhpVersion {
    public function __construct(
        public $version = 'PHP 8.4',
    ) {}

    public function withVersion($version) {
        $new = clone $this;
        $new->version = $version; // ❌ Error!
        return $new;
    }
}
The Elegant Way PHP 8.5 
final readonly class PhpVersion {
    public function __construct(
        public $version = 'PHP 8.4',
    ) {}

    public function withVersion($version) {
        return clone($this, ['version' => $version]);
    }
}

// ✅ Perfect immutability pattern
Game changer for immutable objects

Finally, modify readonly properties during cloning. This unlocks powerful immutable patterns without boilerplate. Build value objects, DTOs, and event sourcing with confidence.

03

Pipe Operator |>

Read RFC →
Nested Hell PHP < 8.5 
$input = ' Some kind of string. ';

$output = strtolower(
    str_replace(['.', '/', '…'], '',
        str_replace(' ', '-',
            trim($input)
        )
    )
);
// Hard to read, harder to debug
Pipeline Heaven PHP 8.5 
$input = ' Some kind of string. ';

$output = $input
    |> trim(...)
    |> (fn($s) => str_replace(' ', '-', $s))
    |> (fn($s) => str_replace(['.', '/', '…'], '', $s))
    |> strtolower(...);
// Read top-to-bottom, think left-to-right
Functional programming, finally natural

Transform data through a series of operations with crystal-clear intent. The pipe operator brings the elegance of functional languages to PHP. Chain operations without temporary variables or deeply nested calls.

04

#[\NoDiscard] Attribute

Read RFC →
Silent Bugs PHP < 8.5 
function getPhpVersion(): string {
    return 'PHP 8.4';
}

getPhpVersion(); // Oops, forgot to use it!
// No warning, no error
Explicit Intent PHP 8.5 
#[\NoDiscard]
function getPhpVersion(): string {
    return 'PHP 8.4';
}

getPhpVersion(); 
// ⚠️ Warning: Return value must be used!
// Or cast to (void) if intentional
Prevent costly mistakes

Mark functions where ignoring the return value is almost certainly a bug. Perfect for builder patterns, resource allocation, or any function with important side effects.

05

Closures in Constant Expressions

Read RFC →
Separated Logic PHP < 8.5 
class Test {
    #[DataProvider('cases')]
    public function testIt($val) {
        // test logic
    }

    public static function cases() {
        for ($i = 0; $i < 10; $i++) {
            yield [$i];
        }
    }
}
Inline Elegance PHP 8.5 
class Test {
    #[CaseGen(static function(): iterable {
        for ($i = 0; $i < 10; $i++) {
            yield [$i];
        }
    })]
    public function testIt($val) {
        // Logic and data together
    }
}
Co-locate logic with usage

Define closures and first-class callables directly in attributes. No more scattered helper methods—keep test data generators, validators, and transformers right where they're used.

06

array_first() & array_last()

Read RFC →
Manual Iteration PHP < 8.5 
$php = [
    'php-85' => ['state' => 'upcoming'],
    // more versions...
];

$upcoming = null;
foreach ($php as $v) {
    if ($v['state'] === 'upcoming') {
        $upcoming = $v;
        break;
    }
}
Functional Clarity PHP 8.5 
$php = [
    'php-85' => ['state' => 'upcoming'],
    // more versions...
];

$upcoming = array_first(
    array_filter(
        $php,
        fn($v) => $v['state'] === 'upcoming'
    )
);
Ergonomic array access

Get the first or last element without ceremony. Pairs beautifully with array_filter() for expressive data queries. No more manual loops or array pointer manipulation.
